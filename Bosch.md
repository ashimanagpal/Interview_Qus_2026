Bosch Interview Questions:


. How many users were there for that application?

. Suppose we have an application where millions of records need to be retrieved via API and shown in a table. How would you handle it? What recommendations would you give to the backend developer for handling millions of records? What parameters would you ask the backend team to expose in the API? How would you handle filtering and sorting in such a table?

. If the table is sorted by username, what information would you send to the API and how would you handle it?

. How would you design a generic reusable table component in React that can be used across different screens?

. Suppose your product is used by different customers and each customer wants their own company color theme. How would you implement theming across the application?

. How would you pass data between pages/components?

. What is prop drilling? When would you advise against using prop drilling?

. Suppose you have user data that needs to be shown in a User Details component. Would you pass it through all components or handle it differently? In a User Details page, how would you pass and manage the user data?

. Suppose there is a table that needs to be filtered based on a text search (e.g., username). How would you handle the text input and filtering?

. Have you heard of identity management tools like Keycloak?

. After login, how would you handle the authentication token? How would you pass the token to every API request?

. How would you handle API errors globally?

. Suppose the token expiry time is 3 minutes. How would you refresh the token? Would you call the refresh token API after 3 minutes or before the token expires?

. Suppose you have a dashboard with 20+ widgets, and each widget has its own API call and renders charts/data. How would you handle performance, rendering, API calls, and memory consumption?
